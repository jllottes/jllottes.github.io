Sometimes we want to zoom in on a particular part of our plot. We can set $x$-limits and $y$-limits using the plt.xlim() and plt.ylim() commands.

plt.xlim(a,b) will restrict the plot to where $a \leq x \leq b$.

plt.ylim(c,d) will restrict the plot to where $c \leq y \leq d$.

The Collatz Conjecture¶

Consider the function $$ f(n) = \begin{cases} 3n+1 & \text{if $n$ is odd} \\ n/2 & \text{if $n$ is even} \end{cases} $$

For any starting $n$, we generate a sequence $\{a_i\}$ where $$ a_0 = n, \qquad a_{i+1} = f(a_i) \quad \text{for i = 0,1,2,}\cdots $$

In [15]:
def f(n):
    if n%2 == 1:
        return 3*n+1
    else:
        return n//2

The Collatz conjecture states that for any starting positive integer $n$, the sequence eventually reaches 1.

Exercise. Define a function collatz(n) that generates the sequence of values $\{a_i\}$ where $a_0 = n$ and $a_{i+1} = f(a_i)$, stopping when the sequence reaches 1.

In [24]:
def collatz(n):
    orbit = [n]
    while True:
        ai = orbit[-1] # most recent number in the sequence
        fai = f(ai)    # find the next entry in the sequence
        orbit.append(fai)
        if fai == 1:
            break
    return orbit

We can clean this up slightly:

In [25]:
def collatz(n):
    orbit = [n]
    while not orbit[-1] == 1:
        orbit.append(f(orbit[-1]))
    return orbit

Project 2 - Pythagorean triples¶

see course webpage

A tuple of integers $(a,b,c)$ is a Pythagorean triple if $a^2 + b^2 = c^2$.

Let's start with a function that takes in an integer and checks if it is a perfect square.

In [36]:
from math import sqrt

def is_square(n):
    if sqrt(n) == round(sqrt(n)):
        return True
    else:
        return False
In [37]:
is_square(4)
Out[37]:
True

This looks like it's working, but it's always risky to check whether two floats are equal to one another. Another approach that is slightly safer:

First, compute $\sqrt{n}$. Then take the integer part of $\sqrt{n}$. Then square the result and check if it is equal to $n$.

In [38]:
def is_square(n):
    if int(sqrt(n)) ** 2 == n:
        return True
    else: return False
In [39]:
is_square(4)
Out[39]:
True

There is still a possible complication if Python evaluated something like sqrt(9.0) = 2.99999999. We can get around this by adding say 0.5 to sqrt(n) before truncating to make up for any slight under estimation of the square root.

In [40]:
def is_square(n):
    if int(sqrt(n) + 0.5) ** 2 == n:
        return True
    else: return False
In [41]:
is_square(9)
Out[41]:
True
